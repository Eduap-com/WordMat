Двоичные файлы, включенные в данный дистрибутив
-----------------------------------------------

Дистрибутив Maxima для Windows включает ряд двоичных
файлов из других свободных программных проектов, 
которые тоже базируются на Sourceforge.


gcc: 

gcc.exe, cc1.exe и файлы в поддиректориях 
lib/gcc-lib и include/ из mingw версии gcc.  
Они доступны на http://prdownloads.sf.net/mingw/
     

binutils:

as.exe из mingw (http://www.mingw.org/) порта binutils,
доступного на http://prdownloads.sf.net/mingw/


gnuplot:

Файлы wgnuplot.exe, wgnuplot.hlp и wgnuplot.mnu из Windows 
дистрибутива gnuplot, доступного на http://gnuplot.sourceforge.net


wxMaxima:

Файлы в поддиректории wxMaxima из Windows дистрибутива
wxMaxima, доступного на http://wxmaxima.sourceforge.net


Графическая оболочка Maxima и брандмауэры
-----------------------------------------

Иногда графическая оболочка Maxima (xmaxima или wxMaxima)
не может запустить Maxima, или сообщает о превышении времени ожидания, 
или не может получить результат выполнения команд Maxima.
Весьма вероятно, что данная проблема вызвана установленным в
системе брандмауэром и/или антивирусным программным обеспечением.
Графическая оболочка Maxima взаимодействует с вычислительным
ядром посредством сокетов.  Брандмауэр и/или антивирус могут
блокировать данное взаимодействие, т.к. некоторые вредоносные
программы тоже используют сокеты.

Для устранения проблемы:

1.  Найдите панель управления брандмауэра и/или антивирусной
    программы. 

2.  Найдите графическую оболочку GUI в списке заблокированных программ
    и разблокируйте её.  Графическая оболочка Maxima может числится
    в этом списке под именем "Tcl/Tk" (имя инструментального пакета,
    используемого программой xmaxima).


Предотвращение выполнения данных (DEP)
--------------------------------------

Иногда не работает не только графическая оболочка Maxima, 
но также и версия Maxima для командной строки 
(maxima.bat запускается и мгновенно завершается).
Весьма вероятно, что данная проблема вызвана механизмом
предотвращения выполнения данных (Data Execution Prevention, DEP).
Некоторые реализации Lisp выполняют код в областях памяти,
предназначенных для хранения данных.  Встроенный в Windows
механизм DEP блокирует работу таких программ, т.к. некоторые
вредоносные программы могут тоже исполняют код в области
данных.

Для устранения проблемы:

  Включите исполнимый файл Maxima, вместе с полным путем к нему, например
  C:\Program Files\Maxima-5.12.0\lib\maxima\5.12.0\binary-gcl\maxima.exe
  к списку исключений DEP  (Панель управления -> Система -> Дополнительно -> 
  Быстродействие -> Предотвращение выполнения данных).


Недостаточно памяти для переменных среды
----------------------------------------

При попытке запуска maxima на машине с Windows 9x возможно 
получение сообщения об ошибке

    Недостаточно памяти для переменных среды (Out of environment space)

Статья 230205 из базы знаний Microsoft (Microsoft Knowledge Base)
(http://support.microsoft.com/support/kb/articles/Q230/2/05.ASP)
поможет исправить проблему.

ПРИЧИНА

Данная проблема возникает, если в среде окружения
MS-DOS недостаточно  памяти для того, чтобы установить 
значение переменной среды. 

СПОСОБ ИСПРАВЛЕНИЯ

Для разрешения проблемы необходимо увеличить размер среды окружения,
доступной программам MS-DOS. Для этого можно использовать два следующих
метода. 


Изменить размер среды окружения для Всех программ MS-DOS

Чтобы увеличить размер среды окружения для всех выполняемых в Windows 
программ MS-DOS необходимо отредактировать команду Shell в файле Config.sys.
Для этого следует выполнить следующие шаги: 

1.  Нажмите Пуск и выберите Выполнить...

2.  В открывшемся окне введите sysedit и нажмите OK. 

3.  Выберите окно Config.sys. 

4.  В начале строки Shell= введите REM и нажмите клавишу ПРОБЕЛ. 

    Если строка Shell= отсутствует, перейдите к следующему шагу. 

5.  Нажмите клавишу HOME. 

6.  Для создания новой строки Shell= line введите нижеследующее 
    и нажмите ENTER: 

      SHELL=C:\COMMAND.COM /E:4096 /P 

7.  В меню Файл (File) нажмите Сохранить (Save). 

8.  В меню Файл (File) нажмите Выход (Exit). 

9.  Перезагрузите компьютер. 


Изменить размер среды окружения для определенной программы MS-DOS

Чтобы увеличить размер среды окружения для maxima.bat, следует
выполнить следующие шаги:

1.  Щелкните правой кнопкой мышки на ярлыке maxima.bat, и выберите Свойства. 

2.  Выберите закладку Память. 

3.  В окошке Переменные среды введите число килобайт (KB), которые требуются
    для работы программы, и нажмите OK. 

ЗАМЕЧАНИЕ: Максимальное количество памяти, которое может быть выделено, равно 4096 KB.
